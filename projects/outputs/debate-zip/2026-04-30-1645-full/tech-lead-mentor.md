# Tech Lead Mentor — 분석

## 한 줄 요약

기획·도메인은 단단하지만, **분석 탭을 핵심으로 끌어올린 4-30 결정으로 인해 3개월 짜리 프로젝트가 6개월 짜리로 부풀어 있는 상태**이며, 이대로면 중간 발표일에 "실시간 STT까지는 됨, 분석은 목업" 수준에서 멈출 가능성이 매우 높다.

## 렌즈 A: Shipping Lens (3개월 중간 발표까지)

### 3개월 내 "실사용 가능한 완성도"로 중간 발표까지 도달 가능성: **낮음**

근거:

1. **현재까지 8주(4-14 → 4-30) 동안 회의·기획·인터뷰 설계만 진행**되었고, 본격 구현은 아직 시작도 안 했다. 잔여 기간이 3개월이라면 UT(2주) + 발표 준비(2~3주)를 빼고 실 구현 약 7~8주가 남는다. 기능 핵심 3가지(실시간 STT 요약 / 분석 탭 / 산출물)에 7~8주를 분배하면 기능당 2주 안 된다. 신OO 멘토가 "기능마다 2주 정도면 괜찮다"고 한 말은 **블랙박스 영역을 멘토가 잡아주는 전제**가 깔려 있다는 점에 주의해야 한다 — 멘토는 항상 옆에 있지 않다.

2. **4-30 회의에서 분석 탭이 실질적 핵심으로 격상**되었다(황OO 멘토 "분석 탭에 좋은 결과가 뽑혀야 들어온다", "매력 기능: 분석 탭"). 그런데 분석 탭에 박힌 항목 수를 세면:
   - 팀별 분석: LLM Judge(쟁점별 우세/결정적 순간) + 팀 성과 지표(실시간 승리 확률 / 종합 설득력) + 팀 논증 구조(주장 트리/근거 유형/일관성/공격수비 분류) + 발화 점유율 = **약 12개 sub-feature**
   - 개인별 분석: 종합 평가 + 논증 능력(논리 오류 탐지/근거 품질/반박 직접성/질문 유형) + 발화 통계(WPM/필러/TTR/어휘 수준) + 상호작용(말끊기/응답 지연/지칭) + 화법(단정/헷지/비유) = **약 18개 sub-feature**
   - 합계 30개 항목. 3개월짜리 4인 프로젝트 분량이 아니다. **이건 12개월짜리 풀 분석 플랫폼의 백로그다.**

3. **신OO 멘토가 "토론 요약은 결국 사람이 봐야 알 수 있다"는 품질 측정 문제를 명확히 지적했고**, 황OO 멘토가 그 답으로 "사람 정답셋 vs LLM 결과 비교"를 제시했다. 이건 **프롬프트 1번 돌리고 끝나는 문제가 아니라, 정답셋 구축에 인터뷰 토론 영상 10개+ 수작업 라벨링이 필요한 작업**이다. 인터뷰 일정·토론 동아리 컨택이 4-30 시점 아직 미정인 상태에서, 정답셋 구축 시간을 일정에 반영했는지 회의록상 보이지 않는다.

### UT 기간 확보 여부: **부분 명시 (위험)**

- 회의록에 "유저 인터뷰 4회 온라인 + 1회 오프라인 + 설문 4회"라는 양적 계획은 있다(예산 30만원, 기간 ~9.20).
- 그러나 이건 **인터뷰 일정**이지 **UT(완성된 제품을 실제 토론에 투입하는 사용자 테스트)**가 아니다. 4-23 회의록 D-03 "동아리 1개 확보 → 2주 간격 MVP 데모·피드백 루프"가 UT에 가까운데, 4-27/4-30 회의에서 **"지금 당장 제휴를 맺지 말고 필요한 경우에 단발성으로 컨택"**으로 후퇴했다. 이건 위험한 후퇴다. 단발성 인터뷰만으로는 "실 사용자가 쓸 수 있는 완성도" 검증 불가.
- **확인 필요**: 2주 스프린트 기준 파일럿 동아리 1곳 정기 사용 루프가 9월 발표 전까지 몇 회 돌아가는가? 회의록상 명시 없음.

### 발표 준비 기간 확보 여부: **없음**

- 4-30 회의록의 "다음 주 회의 전 TODO"는 모두 목업·기능 항목이다. 발표 준비(데모 시나리오 / 영상 / PT / 5분 데모용 동아리 토론 데이터 캡처)는 어디에도 없다.
- 콜리의 "PPT를 AI로 찍기" 숙제는 4-23 → 4-27까지 이월된 상태다. 이걸 이월하지 말고 발표 준비 캘린더로 빨리 옮겨야 한다.

### 숨은 함정 TOP 3 (3개월 압축 기준)

1. **실시간 STT 파이프라인의 신뢰성 함정**
   - 신OO 멘토가 명시적으로 지적했다: 음질·잡음·keep-alive 안 보장·네트워크 끊김 후 재연결·소켓 라이프사이클. 4-23 회의록의 O-206 "한국어 4인 스트리밍 STT+LLM 1초 내 불가, 준실시간 5~8초 명시"도 같은 맥락. **이건 1주차에 "Whisper로 STT 됨" 데모 만들고 안심하다가 6주차 동아리 데모에서 박살나는 함정이다.** 실측 환경 녹음(노트북 마이크, 4명 둘러앉음, 사이드 잡음)에서 WER이 어떻게 나오는지를 1주차에 잡아야 한다. 이게 50% 이하면 STT 엔진 교체 의사결정을 빨리 내려야 한다.
   - 확인 필요: 한국어 다화자 토론 환경 STT WER 벤치마크는 공개 자료가 거의 없다. 직접 측정 필수.

2. **"실시간 승리 확률(체스처럼)"이라는 4-30 ⭐⭐⭐ 기능의 함정**
   - 회의록 4-1-2에 **"실시간 승리 확률 like 체스" + "종합 설득력 점수"**가 ⭐⭐⭐로 박혀 있다. 이건 하지 마라. 체스 승률은 알파제로급 모델이 수백만 자가대국으로 학습한 결과다. 토론 승리 확률을 LLM으로 실시간 추정하는 건 **(1) 정답셋이 없음 (2) 학습 데이터가 없음 (3) "왜 70%인지" 면접관에게 설명 불가능**이다. 이걸 데모로 보여주면 SWM 평가위원·면접관이 정확히 첫 질문으로 "근거가 뭐예요?"를 묻는다. 4-23 O-102가 이미 같은 지적을 했었는데(자동 스코어링의 책임 범위), 4-30 회의에서 미해결인 채로 ⭐⭐⭐만 늘어난 상태다.
   - **권고: 승리 확률은 지워라. 대신 "쟁점별 우세 판정(사후, LLM Judge)"에 정답셋과 근거 인용을 묶어서 한 카드로 만들어라. 이게 면접에서 답할 수 있는 카드다.**

3. **Server Driven UI + 카드뉴스 에디터 + Floating UI + 실시간 분석 차트의 동시 시도**
   - 황OO 멘토가 프론트 도전 포인트로 언급한 게 너무 많다: SDUI(JSON 스키마 기반 UI) / 카드뉴스 에디터 DnD / 요약 하이라이트 Floating UI / 실시간 SVG vs Canvas 차트. 이걸 **혼자 개발하는 프론트(에프이)가 3개월 안에 다 한다**는 가정은 비현실적이다. 멘토 본인도 "반응형 UX 고려, 프론트 혼자 개발하니까 추천하지는 않지만 하고 싶으면 하기"라고 빠져나갔다. 추천하지 않는 건 빼라.
   - **권고: 프론트 한 명이 3개월에 깊게 팔 수 있는 건 정확히 하나다. 분석 탭의 실시간 차트(Canvas + WebSocket)를 "면접에서 30분 떠들 카드"로 고른다면, SDUI와 카드뉴스 에디터는 잘라내거나 후순위로 미뤄라.**

회의록 시간순 추적에서 명확히 보이는 부풀어오름:

**4-30 회의록 4-1-3 "사후 분석 vs 실시간 갱신"**: "실시간 갱신되면 → 생동감 있다 + 기술적 도전요소가 많다" — **이 줄이 가장 위험한 줄이다.** "기술적 도전요소가 많다"는 이유로 실시간을 고르려는 유혹이 있는데, 3개월 일정에서 분석 탭 실시간 갱신은 STT 파이프라인이 아직 안 잡힌 상태에서 두 배 더 어려워진다. 사후로 못 박아라. 면접에서 "왜 사후로 했냐?"는 질문에는 "STT 안정성 우선이라는 이OO 멘토 조언과 정합성 우선 원칙 때문"이라고 답하면 된다.

## 렌즈 B: 취업 레버리지

### 면접 30분 서사가 될 기술 깊이: **부분 (분기점)**

- **있음 (지금 살릴 수 있는 카드)**:
  - 실시간 다화자 STT 파이프라인 + 소켓 라이프사이클 관리(끊김/재연결/버퍼링) — 백엔드 면접 30분 카드 1개. 신OO 멘토가 정확히 이 지점을 강조한 게 큰 자산이다.
  - 실시간 분석 차트(Canvas/SVG/DOM 트레이드오프 의사결정) — 프론트 면접 30분 카드 1개. 황OO 멘토 답변이 면접 답변으로 거의 그대로 재활용 가능하다.
  - 토론 요약 LLM 품질 측정 정답셋 구축 + LLM-as-Judge 평가 파이프라인 — AI/백엔드 공통 카드 1개. "AI 산출물 품질을 어떻게 측정했나?"는 2026년 시장에서 거의 모든 AI 통합 회사가 묻는 질문이다(확인 필요, 단 LLM-as-Judge 패턴 자체는 업계 표준화된 패턴).

- **부분 (지금 카드로 안 됨)**:
  - 4-30에 박힌 ⭐⭐⭐ 기능들은 "이걸 만들었다"고 말할 수는 있지만, **"왜 이 방식이냐"**라는 후속 질문에 답할 근거가 부족하다. 실시간 승리 확률, 종합 설득력 점수, 논리적 오류 탐지(허수아비/성급한 일반화 등 LLM 분류) — 모두 **검증 가능한 정답이 없는 작업**이다. 만들 수는 있어도 면접에서 "정확도를 어떻게 측정했어요?"라고 물으면 답할 게 없다.

- **없음**:
  - 대용량 트래픽·분산 시스템 카드는 없다. 이건 이 프로젝트 성격상 어차피 만들기 어렵다. 받아들이고 다른 카드를 강화하라.
  - DevOps/인프라 카드는 회의록 어디에도 없다. AWS 락 멘토 언급 외에는. 백엔드(쿼찬?) 담당이 인프라까지 가져갈지가 4-23 O-201에서 미해결인데, 4-27/4-30 회의록에서도 해결되지 않았다. 이게 안 잡히면 백엔드 면접 카드가 STT 파이프라인 1장으로 끝난다.

### 시장 수요 정렬

(의견 — **확인 필요** 표기)

- **AI/LLM 통합 + 실시간 처리**: 2026년 신입 시장에서 수요 있는 조합으로 보인다. 단, **"LLM 래퍼" 비판**(4-23 O-205에서 자체 인지)을 어떻게 면접에서 방어할지가 관건. "정답셋 만들어서 평가했다"가 그 답이다.
- **STT 파이프라인 + 소켓**: 2026 시장 수요는 보통(확인 필요). 단 "실시간 처리 + 끊김 복구 + 정합성"이라는 시스템적 깊이로 묶으면 백엔드 카드로 강하다.
- **실시간 차트(Canvas/SVG)**: 데이터 시각화 수요는 핀테크·관제·헬스케어에서 안정적으로 있다(확인 필요). 변환 가능한 카드.
- **카드뉴스 에디터(DnD/SDUI)**: 독립적으론 약하다. 에디터 만든 사람은 많다. 단 SDUI를 "왜 도입했나"의 의사결정 서사가 있으면 미드 레벨까지는 통한다.

회의록에서 식별된 팀원 4명: 콜리(기획·도메인) / 에프이(프론트) / 커찬(?) / 쿼찬(백엔드 추정)

**현재 지분 평가**:

- **콜리**: 도메인·기획·인터뷰·PT 모두 본인 자산. 단, 4-23 N-04 "콜리 1인 기획 단일장애점", 4-23 O-209 "Bus Factor"에서 본인이 인정한 리스크가 4-30까지 미해결. **취업 카드로 보면 콜리는 "PM 역량"으로 풀어야 하는데, 백엔드/프론트 신입 채용에는 PM 역량이 1차 합격 사유가 안 된다.** 콜리는 본인 코드 영역을 명확히 가져가야 한다. 회의록 어디에도 콜리의 "내가 짤 코드"가 없다.
- **에프이**: 4-27 메모에 본인 욕심(DnD, 에디터, 카드뉴스)이 명확히 있고, 4-29 황OO 멘토 미팅에서 도전 포인트(실시간 차트, SDUI, Floating UI)도 풍부하게 가져왔다. **단 욕심이 4개라서 1개로 좁혀야 한다.** 권고: 실시간 분석 차트(Canvas 선택 의사결정) + 분석 탭 페이지 전체. 카드뉴스 에디터는 후순위.
- **커찬**: 4-27 회의록에서 "[기술 관점]" 메모를 직접 작성하며 라이브 요약 기능을 BE/FE 핵심 키로 본다는 본인 의견을 냈다. **하지만 BE인지 FE인지 회의록에서 명시되지 않는다.** 4-23 O-201이 "쿼찬 합류 확인" 후 분담이라고 적혀 있어 헷갈리는데, 4-27/4-30까지도 명시 없다. **이건 4-30 회의 다음 회의 1순위 어젠다여야 한다.**
- **쿼찬**: 4-23 O-209에서 "쿼찬 부재 / 합류 의사 미확인"으로 명시됐는데 4-27/4-30 회의록에서 본인 발화/의견이 0건. **합류 했는지조차 분석 자료에서 확인 불가**. 이게 사실이라면 프로젝트 자체가 4-23부터 1주에 1명씩 줄어드는 중이다.

**판정**: 4명 중 4명이 자기 1인칭으로 면접 답할 수 있는 상태가 아니다. **현재 콜리·에프이 2명이 면접 카드를 가져가고, 커찬·쿼찬은 카드 부재 상태**라고 봐야 한다. 이건 가장 큰 취업 리스크다.

### "왜 A 대신 B" 결정 포인트

현재 명확하게 형성된 의사결정 서사:

1. **세다(아카데미식) 토론 vs 자유 토론** → 세다 선택. 근거: 턴 구조로 화자 분리 자동, DER 우회. **(좋은 카드. 면접에서 이대로 답하면 됨)**
2. **버티컬(토론 도메인) vs 범용 회의록** → 버티컬 선택. 근거: 클로바노트와 차별화, 도메인 인사이트 누적. **(기획·PM 카드. 백엔드/프론트 면접에선 약하지만 자기소개에 쓸 만함)**
3. **STT를 FE vs BE** → 미정. 신OO 멘토 답변(BE 권장)도 있고 황OO 멘토 답변(FE 직접도 가능)도 있어 **결정이 안 났다**. 4-30 회의록에 미해결.
4. **사후 분석 vs 실시간 갱신** → 미정. 4-30 회의록에 명시적으로 양쪽 다 열어둠.

**부족한 결정 카드**:

- 위 3, 4가 미정인 채로 다음 주 회의에서 목업만 만들면, 결국 "코드 짤 때 결정" 모드가 되어 의사결정 서사가 면접에 못 쓰게 된다. **결정 → 근거 기록 → 추후 회고**의 사이클을 의식적으로 운영해야 한다.

## 렌즈 충돌 지점

이 팀에서 명확히 충돌하는 두 압력:

1. **Shipping 압력 vs 면접 깊이 압력 (분석 탭)**
   - Shipping 관점: 분석 탭의 30개 항목은 잘라내야 3개월 일정에 맞다. ⭐⭐⭐ 절반은 후순위로.
   - 면접 관점: 분석 탭이 매력 기능이고(황OO 멘토 인증), 프론트 실시간 차트 카드의 본진이다. 잘라내면 면접 카드가 약해진다.
   - **합의안**: 분석 탭 항목을 **3장**으로 좁힌다. (1) 사후 LLM Judge 쟁점별 판정 + 근거 인용 (백엔드/AI 카드), (2) 실시간 차트(발화 점유율·턴 길이·논증 트리 — 진짜 실시간 누적되는 통계만, Canvas 깊게 파기) (프론트 카드), (3) 개인 종합 평가(베스트/워스트 발언 하이라이트, 정답셋 비교) (AI/백엔드 카드). 나머지 항목은 "확장 영역"이라고 라벨링하고 후순위로 깡그리 옮겨라.

2. **Shipping 압력 vs 면접 깊이 압력 (STT 위치)**
   - Shipping 관점: 신OO 멘토 권고(BE)가 안전. 안정성 우선.
   - 면접 관점: FE에서 처리하면 "왜 FE에 두고 어떻게 백엔드와 동기화했는가"라는 깊은 카드 가능. 단 망가지면 데모 자체가 안 됨.
   - **합의안**: BE에서 STT 처리 → BE에서 정합성·재연결·버퍼 관리 → FE는 WebSocket 구독자. 이게 신OO 멘토 권고와 정합성 원칙에 맞다. 면접에서는 "왜 FE 아닌 BE인가"의 의사결정 자체를 카드로 쓰면 된다.

3. **타깃 검증 압력 vs 발표 임팩트 압력**
   - 4-23 회의에서 "동아리 1개 확보 → 2주 간격 데모 루프"가 있었는데, 4-30 회의에서 "단발성 컨택"으로 후퇴. 발표 임팩트로 보면 단발성 인터뷰 5개보다 정기 사용 동아리 1개가 훨씬 강한 데모 자료가 된다. **이 후퇴를 다시 정정할 가치가 있다.** 4-30 컨택 리스트 3명 중 1명을 정기 파일럿으로 깊게 묶어라.

## 권고

### 지금 당장 잘라낼 것

1. **분석 탭 항목 30개 중 25개**. 살릴 3장: 사후 쟁점별 LLM Judge / 실시간 발화 점유·턴·논증 트리 차트 / 개인 베스트·워스트 하이라이트. 나머지(승리 확률, 종합 설득력 점수, 논리적 오류 분류, WPM/TTR/필러/말끊기/응답 지연/지칭 방식/단정 vs 헷지/비유/문장 복잡도)는 모두 후순위.
2. **카드뉴스 에디터 DnD**. 카드뉴스는 자동 생성 결과물 이미지 다운로드만 남기고 에디터는 통째로 잘라라. 에프이의 면접 카드는 분석 탭 차트로 충분.
3. **실시간 승리 확률(체스처럼)**. 정답셋 없는 기능. 만들면 면접관이 정확히 그 지점을 찌른다.
4. **자유 토론 지원, 카드뉴스 인스타 업로드 연동, 학생 PDF 포트폴리오, Role-based View 분기 (대회/교사/동아리)**. 4-23~4-30 사이에 등장한 추가 스코프 전부.

### 지금 당장 더 깊게 파야 할 것 (딱 2개만)

1. **실시간 STT 파이프라인의 안정성 + 정합성**. 1주차 안에 한국어 4인 토론 환경 WER 측정. Whisper(local) / Clova / Daglo 3종을 토론 샘플 5개로 비교. 결과를 의사결정 문서로 남겨라(이게 면접 카드가 된다). 끊김 재연결 시나리오는 시뮬레이션 테스트로 코드에 박아둬라.
2. **LLM 요약·판정 품질 측정 파이프라인**. 토론 영상 10개+ 사람 정답셋 → LLM 결과 → LLM-as-Judge 비교. 이걸 CI에 올려서 "프롬프트 바꾸면 점수 어떻게 변하는지" 측정 가능하게 만들어라. 이게 2026 AI 통합 회사 면접 카드 1순위(확인 필요, 단 LLM-as-Judge가 업계 표준 패턴이라는 점은 사실).
