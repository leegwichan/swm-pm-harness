## 반(反) — 충돌 지점

### 프리셋 충돌축 1 — tech-lead-mentor ↔ peer-competitor (안전한 MVP vs 차별화 도전)

- **tech-lead-mentor**: 분석 탭 30개 항목을 3장으로 컷하고, 카드뉴스 에디터·실시간 승리 확률·Role-based View 분기·자유 토론 등 4-23~4-30 사이 부풀어 오른 스코프 전부를 잘라야 3개월 안에 돌아간다. 사후로 못 박고 안전 스택 우선.
- **peer-competitor**: 차별화 한 줄이 GitHub 양산형(`real-time meeting transcription` + 도메인 명사) 그대로다. 진짜 차별점인 "세다 토론 한정·턴 기반·화자 분리 우회"가 README 첫 줄에 안 들어가 있다. mock 수치 박힌 시안만으로는 30초 후킹 안 됨, 실측 데이터·데모 GIF 필요.
- **충돌 본질**: tech-lead가 "줄여서 살아남자"고 하면 peer는 "그렇게 줄이면 양산형으로 보인다"고 한다.
- **취업 관점 중재**: 차별화 축을 **딱 1개**만 살린다. 그게 "한국어 4명 세다 토론 STT 5~8초 + 주장-근거-반박 트리 추출". 이 1개는 두 페르소나 모두 살리라고 한다(tech-lead의 "더 깊게 팔 2개" + peer의 "블로그 클릭 받는 2·3번 인사이트"가 정확히 이 지점). 나머지(승리 확률·논리 오류 4종·카드뉴스 에디터·자유 토론·Role-based View)는 잘라내고, 잘라낸 것을 README에 "out of scope"로 명시해서 차별화로 환전한다.

### 프리셋 충돌축 2 — target-user-persona ↔ swm-reviewer (유저 단순함 vs 평가위원 난이도)

- **target-user-persona (학부 토론 동아리 부원 박OO)**: 한 화면에 점수·그래프·카드·인용·세특·카드뉴스가 다 있어서 "내가 토요일에 이걸 왜 켜?"가 안 잡힘. 정확도 70% 미만 1회 사고 → 신뢰 0. 분석 탭이 박사급. 3개로 줄이지 않으면 1초 보고 닫음. 회원가입 3단계도 못 넘김.
- **swm-reviewer**: 분석 18개를 3개로 컷해야 평가 통과. 단 카드뉴스/세특을 빼는 것까지 검토하라(법규 리스크, 면접 카드와 무관, 4-29에 본인들도 "정말로 필요한가" 의문). "라이브 동작 데모 vs 영상 합성/모의 데이터"의 갭이 데모 당일 노출됨.
- **충돌 본질**: 유저는 "AI 점수 매기기" 자체를 거부하고 단순함을 원함. 평가위원은 "무엇으로 측정·검증했는가"의 기술 난이도를 원함. UX의 단순함과 평가 난이도가 같은 화면에서 충돌.
- **취업 관점 중재**: 레이어 분리 — 유저 화면(Live·요약 1페이지)은 단순화하고, **분석 탭은 "토론 끝난 후 30분 뒤 메일/카톡으로 전송되는 사후 리포트"**로 옮긴다. 유저는 Live + 사후 리포트 1장을 보고, 평가위원에게는 그 사후 리포트의 백엔드 파이프라인(정답셋·LLM-as-Judge)을 보여주면 된다. 백엔드 지원자는 "사후 리포트 정확도 측정 파이프라인" 카드를, 프론트 지원자는 "Live의 5~8초 지연 UX 트릭 + 사후 리포트 1종의 차트 깊이"를 각자 자기 면접 카드로 가져간다.

### 프리셋 충돌축 3 — tech-lead-mentor ↔ swm-reviewer (현실적 범위 vs 도전적 범위)

- **tech-lead-mentor**: 사후 분석으로 못 박아라. STT를 BE에 두고 안정성·정합성 우선. FE는 분석 탭 차트 1종만 깊게.
- **swm-reviewer**: 분석을 사후로 빼는 것 자체에는 동의하지만, "실시간성"이 차별점이라고 본인들이 박은 이상 **실시간 자막 + 사후 분석**의 양쪽이 데모 당일 모두 살아야 함. 라이브 환경에서 평가위원이 직접 1분 발화하겠다고 하면 그게 라이브 검증.
- **충돌 본질**: tech-lead는 "사후가 안전" / swm은 "실시간 자막은 라이브로 안 돌아가면 차별점 자체가 가짜". 두 페르소나 모두 분석은 사후로 동의하나, **자막의 라이브 보장 수준**에서 갈린다.
- **취업 관점 중재**: 실시간 자막은 5~8초 지연을 명시하고 라이브로 박는다(이게 백엔드 카드의 본진 = 신OO 멘토가 강조한 소켓 라이프사이클·재연결·정합성). 분석 탭은 사후로 박는다. **"실시간 자막 + 사후 분석"이라는 두 단어 구조를 회의록에 못 박고, 그 둘 사이의 경계선을 "토론 종료 시점"으로 정의**하면 두 페르소나 모두 만족.

### 프리셋 외 충돌

- **tech-lead-mentor ↔ peer-competitor: 디자인 컨셉**
  - tech-lead는 디자인 컨셉을 거의 언급 안 함(시간이 그쪽으로 빠진 게 문제라고 평).
  - peer는 "04-23 Tailwind 데모 톤 vs 04-30 에디토리얼 매거진 톤"이 따로 노는 게 약점이라며 04-23 버리고 04-30으로 통일 권고.
  - 취업 관점 중재: 04-30 톤(Fraunces+IBM Plex KR+oklch)으로 통일하되, 통일 자체가 본질이 아님 — 디자인 결정에 들이는 시간을 데이터 파이프라인에 옮기는 게 우선.

- **target-user-persona ↔ peer-competitor: 카드뉴스의 운명**
  - target은 "이 서비스가 살아남는다면 그건 동아리 회장의 카드뉴스 자동화 도구로서일 가능성이 높다 — 핵심을 거꾸로 잡고 있는 느낌"이라고 함.
  - peer는 "카드뉴스 5장짜리 정사각 미리보기"를 30초 후킹 요소로 인정.
  - tech-lead·swm은 카드뉴스를 후순위/잘라낼 항목으로 분류.
  - 취업 관점 중재: **카드뉴스는 살리되 에디터는 자른다.** 자동 생성 결과물 1세트(템플릿 고정형)만 남기고 DnD 에디터·인스타 업로드 연동은 잘라낸다. target이 지적한 "동아리 회장 트리거"의 가치는 인정하되, 본 프로젝트의 면접 카드는 카드뉴스가 아니라 분석 파이프라인이라는 것을 분명히.

### 위험 관리

- **취업 임팩트에 가장 큰 위험**: 쿼찬 합류 미확인 + 커찬 BE/FE 분담 미정. 4-23 O-201부터 보름째 미해결. 이 채로 6주 흐르면 BE 담당자가 자기 면접 카드를 못 만든 채로 발표일을 맞이.
- **완화 방법**:
  1. 다음 회의 30분 1순위로 쿼찬 합류 yes/no + BE 책임자 1인 명시 문서화. yes/no 어느 쪽이든 결정해서 4-23 O-201을 닫는다.
  2. 만약 쿼찬 부재면 SWM 운영부에 팀 변경 신고 + 콜리/커찬 중 1인이 BE를 명시적으로 흡수.
  3. 콜리도 본인의 코드 영역 1개를 박는다(도메인 인사이트만으로는 백엔드/프론트 신입 면접 카드 안 됨).
- **2차 위험**: DT(디베이트 타이머) 소유권 이전 가능성(N-01). 면접에서 "이거 콜리 회사로 이전됐어요"라고 답해야 하면 본인 지분 소실. 소마 발표 전 팀 내부 합의문 1장 필수.
- **3차 위험**: 세특 자동 생성의 교육부 가이드 위반(O-103). UI 카피를 "자동 생성"이 아닌 "근거 확인 기반 초안 보조"로 1주 안에 고정.

## 팀원별 취업 레버리지 점검

회의록에서 식별 가능한 4명: 콜리(기획·도메인·인터뷰), 에프이/FA(프론트), 커찬(?), 쿼찬(BE 추정·합류 미확인). 각자 백/프 지원 루트는 **회의록 어디에도 명시 없음**. 따라서 추정 + "다음 회의에 명시 필요"로 표기.

| 팀원 역할 | 지원 루트(백/프) | 이 프로젝트에서의 지분 | 면접 카드 (본인이 쓸 수 있는 것) | 부족한 것 |
| --- | --- | --- | --- | --- |
| 콜리 | 미명시(추정: 풀스택/PM·도메인) | 도메인·인터뷰·PT·DT 유저풀 | 토론 도메인 STP / 50분 교사 인터뷰 3세그먼트 / 외부 세특 업체 교체 사건 / 세다 토론 한정 의사결정(R-08) | **본인이 짤 코드 영역 0건.** 백엔드/프론트 신입 면접에 PM 카드만으로는 부족. 본인이 소유할 코드 영역 1개 명시 필요. DT 소유권 합의문도 미작성. |
| 에프이/FA | 미명시(추정: 프론트) | 프론트 단독, 04-30 디자인 톤, 멘토 조언으로 차트·SDUI·Floating UI·DnD 등 도전 영역 4개 후보 | 04-30 에디토리얼 톤(Fraunces+IBM Plex+gold) / Canvas vs SVG vs DOM 차트 의사결정 / 5~8초 지연 UX 트릭 / Server Driven UI(만약 살린다면) | 도전 영역 4개를 1~2개로 좁혀야 함(권고: 실시간 차트 + Live UX). 멘토 답변을 자기 언어로 재서술 필요 — 면접에서 "멘토님이 그러셨어요"는 깎임. |
| 커찬 | 미명시(BE/FE 양쪽 흔적) | 4-27 회의에 본인 [기술 관점] 메모 작성. 라이브 요약을 BE/FE 핵심 키로 보는 본인 의견 있음 | (BE라면) STT 파이프라인 보조 / (FE라면) 분석 탭 일부 — 둘 다 명시 안 됨 | **분담 미정 = 면접 카드 0장.** 다음 회의 즉시 BE/FE 명시 필요. |
| 쿼찬 | 미명시(BE 추정) | **4-23 O-201부터 4-30까지 회의 발화/의견 0건. 합류 여부 불명.** | 없음 (현재 상태) | 합류 yes/no 즉시 결정. yes면 STT 파이프라인 + LLM 평가 파이프라인 책임자로 명시. no면 팀 인원 변경 신고 + 다른 1인이 BE 흡수. |

**팀 전체 판정**: 4명 중 **2명(콜리·에프이)만 면접 카드의 단초**가 있고, **2명(커찬·쿼찬)은 카드 부재**. 이게 가장 큰 취업 리스크. 4-23부터 보름째 미해결.

## 다음 회의 전까지 팀이 답해야 할 질문

1. **쿼찬 합류 yes/no를 5월 첫 주 회의 30분 1순위로 결정한다.** yes면 BE 책임자로 명시 + STT/LLM 평가 파이프라인 카드 배정. no면 SWM 운영부에 팀 변경 신고 + 콜리/커찬 중 1인이 BE 흡수. **(취업 임팩트: 가장 큼 — BE 카드 형성 가능성 자체)**

2. **커찬은 BE/FE 어느 쪽인가?** 4-27 회의록 본인 메모는 양쪽 다 언급. 명시적 분담 결정. **(취업 임팩트: 큼 — 본인 면접 카드 영역 결정)**

3. **분석 탭 30개 항목을 정확히 3개로 컷한다.** 권고 3종: ① 실시간 발화 점유율/턴 길이 차트(FE 카드) ② 사후 LLM Judge 쟁점별 우세 판정 + 근거 인용(BE/AI 카드) ③ 개인 베스트·워스트 하이라이트. 잘라낼 25개를 회의록에 명시(평가위원이 4-30과 비교할 때 "잘랐다"가 보이게). **(취업 임팩트: 큼 — 면접 카드의 명료성)**

4. **사후 분석 vs 실시간 갱신을 사후로 못 박는다.** 단, 자막 자체는 5~8초 지연으로 라이브 박는다. "실시간 자막 + 사후 분석"이라는 두 단어 구조를 PRD 1줄에 박는다. **(취업 임팩트: 큼 — 차별점 한 줄 형성)**

5. **STT 처리 위치를 BE로 못 박고, 1주차 안에 한국어 4명 토론 환경 STT WER 실측을 끝낸다.** Whisper(local)/Clova/Daglo 3종 비교, 토론 샘플 5개. 결과를 의사결정 문서로 남김(이게 백엔드 면접 카드 1번이 됨). **(취업 임팩트: 큼 — 카드 1번의 실측 데이터)**

6. **9월 발표일까지 마일스톤 캘린더를 작성하고 회의실 벽에 붙인다.** UT 2주 + 발표 준비 2~3주를 빼고 실 구현 주차를 역산. M1/M2/중간평가/M3/수료 데모데이 일자에 검증 가능한 산출물 1개씩 정의. **(취업 임팩트: 큼 — 면접에서 "프로젝트 일정 어떻게 관리했나"의 답)**

7. **콜리의 코드 영역 1개를 박는다.** 도메인·기획·인터뷰만으로는 백엔드/프론트 신입 면접 카드 안 됨. 권고: 4단계 LLM 파이프라인(유닛 분할/의도 분류/Rebuts 매칭/세특 생성+인용)의 프롬프트 엔지니어링 + LLM-as-Judge 정답셋 라벨링 책임. **(취업 임팩트: 콜리 본인 면접 카드)**
