# analyze-meeting

이 하네스는 팀의 **최우선 목표**를 고정한다:

> **"SWM에서의 활동을 바탕으로 2026년 백엔드/프론트 신입개발자 서비스/대기업 취업에 성공한다."**

모든 종합(합·권고)은 이 목표에 도움이 되는가를 기준으로 판단한다.

## 세부 규약 파일 지도

이 skill의 구체 내용은 아래 파일들로 분리되어 있다. 메인 Claude는 skill 호출 시 **필요한 파일을 `Read`로 불러** 규약을 따른다.

| 파일 | 다루는 내용 | 언제 읽어야 하는가 |
| --- | --- | --- |
| [`invocation.md`](./invocation.md) | 호출 형태, mode, 등록 에이전트 목록 | 항상 — 사용자 입력 파싱 시 |
| [`execution.md`](./execution.md) | 실행 절차 6단계 + 서브에이전트 프롬프트 템플릿 | 항상 — 실제 실행 로직 |
| [`output-format.md`](./output-format.md) | 최종 리포트 파일 포맷 + 개인정보 마스킹 규칙 | 리포트 작성 직전 |
| [`constraints.md`](./constraints.md) | 에러 처리 + 금지 사항 | 항상 — 체크리스트로 사용 |
| `../synthesis-framework/SKILL.md` | 정(正)·반(反)·합(合) 종합 규약 | 종합 단계 진입 시 |

## 호출 로드맵 (요약)

1. 사용자 입력 파싱 → [`invocation.md`](./invocation.md) 참조.
2. 실행 절차 6단계 → [`execution.md`](./execution.md) 참조.
3. 종합 단계에서 `synthesis-framework` skill 로드.
4. 리포트 포맷·마스킹 → [`output-format.md`](./output-format.md) 참조.
5. 에러·금지 사항은 전 과정에서 [`constraints.md`](./constraints.md) 준수.

## 최상위 원칙 (다른 파일에서 반복되지 않음)

- **병렬 호출**: 서브에이전트는 반드시 단일 메시지의 다중 tool call로 **병렬** 호출한다. 한 에이전트의 출력을 다른 에이전트에게 넘기지 않는다.
- **파일 분리 저장**: 한 실행당 `projects/outputs/<project>/{타임스탬프}-{mode}/` 디렉터리 하나를 만들고, 참여 에이전트마다 `{agent-name}.md` 한 파일씩 + 종합 파일 `synthesis.md` 하나를 생성한다. 단일 파일에 모두 합치지 않는다.
- **원문 보존**: 각 에이전트 개별 파일에 응답을 **축약 없이 그대로** 붙인다. 가독성을 위한 편집 금지(마스킹은 예외). `synthesis.md`에는 원문을 복사하지 않고 **링크로만 참조**한다.
- **프로젝트 디렉터리 자동 생성 금지**: 오타로 유령 프로젝트가 생기는 것을 막기 위해 사용자가 명시적으로 `mkdir`한 디렉터리만 다룬다.
- **최우선 목표 정렬**: 합(合) 단계는 반드시 "2026년 백엔드/프론트 신입 취업 성공"을 판정축으로 삼는다.
