# constraints — 에러 처리와 금지 사항

`analyze-meeting` skill 전체에 걸쳐 적용되는 실패 처리 방침과 금지 사항. 매 실행마다 체크리스트로 활용한다.

## 에러 처리

### 입력 단계 실패

- **프로젝트 디렉터리 없음** → `mkdir -p projects/inputs/<project> projects/outputs/<project>`를 안내하고 중단. 자동 생성 금지.
- **입력 파일 없음 / 읽기 실패** → 경로 확인 안내 후 중단. 임의로 다른 파일을 고르지 않는다.
- **mode 인자가 지원 목록에 없음** → 지원 모드 목록을 보여주고 중단.

### 서브에이전트 호출 실패

- 일부 에이전트만 실패했다면: **성공한 에이전트 결과만으로 종합**하되, 리포트 상단 메타데이터 섹션에 "실패한 에이전트: {목록}, 사유: {...}"를 명시한다.
- 전원 실패했다면: 종합하지 않고 실패 사실을 사용자에게 보고한 뒤 중단.
- 에이전트가 페르소나 제약을 위반한 응답을 낸 경우(예: `target-user-persona`가 기술 스택 평가를 시작): **재호출하지 말고** 받은 응답 그대로 정(正) 섹션에 넣는다. 리포트 상단에 "페르소나 이탈 관찰됨"을 기록하고, 사용자에게 해당 에이전트 파일(`.claude/agents/...`)의 튜닝을 권고한다.

### 저장 단계 실패

- 에이전트 개별 파일(`{agent-name}.md`) 저장에 실패하면: 실패한 에이전트 이름과 사유를 `synthesis.md` 메타데이터에 기록하고, 해당 에이전트의 원문을 `synthesis.md` 하단에 "## 저장 실패 에이전트 원문" 섹션을 만들어 붙인다(원문 유실 방지).
- `synthesis.md` 저장에 실패하면: 현재 대화 응답에 종합 리포트 전문을 붙여 출력한다.
- 경로 충돌(동일 분 단위 타임스탬프 디렉터리가 이미 존재)이면 디렉터리명 뒤에 `-2`, `-3` 순차 접미사를 붙인다(파일명이 아니라 **디렉터리명**에 붙인다).

## 금지 사항

### 실행 관련

- 서브에이전트를 **순차** 호출하지 않는다(병렬 필수, [`execution.md`](./execution.md) 단계 3 참조).
- 한 에이전트의 출력을 다른 에이전트에게 **전달하지 않는다**.
- 프로젝트 디렉터리를 **자동으로 생성하지 않는다**(오타로 유령 프로젝트가 생기는 것을 막기 위함).
  - 예외: 출력 디렉터리(`projects/outputs/<project>/`)는 입력 디렉터리가 존재하는 경우에 한해 자동 생성 허용.

### 종합 관련

- 서브에이전트의 **페르소나 톤을 종합 단계에서 세탁하지 않는다**. 예: "드라이한 swm-reviewer가 '잘하고 있다'고 했다"는 종합은 페르소나 오염이다.
- **에이전트 개별 파일에 원문을 요약하거나 축약하지 않는다** — 원문 그대로 붙인다. 가독성을 위한 편집 금지(마스킹만 예외).
- 에이전트 원문을 `synthesis.md`에 **중복 복사하지 않는다**. 링크로만 참조.
- 모드에 참여하지 않은 에이전트 파일은 **생성하지 않는다**. `synthesis.md`의 정(正) 섹션 링크 목록에서도 제외.

### 출력 관련

- 개인정보 마스킹을 서브에이전트 호출 단계에서 **미리** 적용하지 않는다(원문 분석 필요). 저장 직전에만 적용.
- 대화 응답에 전체 리포트를 그대로 붙이지 않는다. 저장 경로와 한 줄 요약만 전달.
- 추측성 기술 사실을 단정하지 않는다. 모르는 것은 "확인 필요"로 표기.

## 실행 전 자기 점검 체크리스트

실제 서브에이전트를 호출하기 전에 메인 Claude는 다음을 확인한다.

- [ ] 사용자 입력에서 `<project>` 인자를 제대로 파싱했는가?
- [ ] `projects/inputs/<project>/` 디렉터리가 존재하는가?
- [ ] 선택된 입력 파일 경로가 확정되었는가?
- [ ] `mode`에 따라 호출할 에이전트 목록이 결정되었는가?
- [ ] 서브에이전트 호출을 **단일 메시지의 다중 tool call**로 구성할 준비가 되었는가?

## 리포트 작성 전 자기 점검 체크리스트

종합 리포트를 저장하기 직전 메인 Claude는 다음을 확인한다.

- [ ] 참여한 **모든 에이전트마다 개별 파일**(`{agent-name}.md`)이 준비되었는가?
- [ ] 각 에이전트 파일에 원문이 **축약 없이** 들어갔는가?
- [ ] `synthesis.md`에 에이전트 원문을 **중복 복사하지 않고** 링크로만 참조했는가?
- [ ] `synthesis.md`의 반(反)·합(合)·팀원별 점검은 `synthesis-framework`의 체크리스트를 통과했는가?
- [ ] 마스킹 대상(이름·전화·이메일)이 **모든 파일**에서 치환되었는가?
- [ ] 저장 디렉터리 경로가 `projects/outputs/<project>/{타임스탬프}-{mode}/` 형식인가?
- [ ] 실패한 에이전트가 있었다면 `synthesis.md` 상단 메타데이터에 기록되었는가?
